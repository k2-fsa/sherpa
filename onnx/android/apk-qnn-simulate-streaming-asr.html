Qualcomm NPU (QNN): APKs for real-time/streaming speech recognition with non-streaming ASR models

This page lists the real-time speech recognition with non-streaming models APKs for sherpa-onnx, one of the deployment frameworks of the Next-gen Kaldi project.
It uses Qualcomm NPU for computation, with HTP backend. You need a Phone with Qualcomm SM8350 chips or more powerful chips, e.g. SM8450, SM8475, SM8550, SM8650, SM8750, SM8850, etc.
The name of an APK has the following rule:
  • sherpa-onnx-{version}-qnn-{arch}-simulated_streaming_asr-{lang}-{len}-{seconds}-{model}.apk
where
  • version: It specifies the current version, e.g., 1.12.17
  • arch: The architecture targeted by this APK, e.g., arm64-v8a, armeabi-v7a, x86_64, x86
  • qnn: It means qnn is used
  • lang: The lang of the model used in the APK, e.g., en for English, zh for Chinese
  • len: Since QNN does not support dynamic input shapes, we limit the maximum duration the model can handle. For example, if the limit is 10 seconds, any input shorter than 10 seconds will be padded to 10 seconds, and inputs longer than 10 seconds will be truncated to that length.
  • model: The name of the model used in the APK

You can download all supported models from https://github.com/k2-fsa/sherpa-onnx/releases/tag/asr-models-qnn

If your are interested in the source code for the APK, please visit https://github.com/k2-fsa/sherpa-onnx/tree/master/android/SherpaOnnxSimulateStreamingAsr

Note about the license The code of Next-gen Kaldi is using Apache-2.0 license. However, we support models from different frameworks. Please check the license of your selected model.
